# LLM Wiki Schema & Rules

이 파일은 AI 에이전트(저)가 이 지식베이스(`c:\Python\wiki`)를 유지보수할 때 참조하는 핵심 규칙 문서입니다.
지식 구조와 포맷, 작업 절차가 명시되어 있습니다.

## 1. Directory Structure

- `raw/`: 사용자가 직접 저장하는 원본 소스가 위치합니다. 이 폴더의 파일은 AI가 절대 수정하지 않습니다. (Read-Only)
- `pages/`: AI가 원본을 분석하여 생성한 가공 문서들(요약, 개념, 인물, 기술 등)이 위치합니다.
- `index.md`: 모든 문서의 리스트와 요약이 카테고리별로 기재되는 메인 허브입니다.
- `log.md`: AI의 모든 작업 내역이 시간순으로 기록되는 문서입니다.

## 2. Operation Workflows

### 2.1 Ingest (문서 수집 및 분석)
사용자가 새로운 문서를 `raw/` 폴더에 넣고 분석을 요청하면, AI는 다음 절차를 따릅니다.
1. 문서를 읽고 핵심 내용을 추출합니다.
2. `pages/` 폴더 내 기존에 관련된 문서가 있는지 확인하고, 필요하다면 기존 문서를 업데이트합니다 (새 정보 추가 또는 충돌 데이터 반영).
3. 새로운 개념이나 요약이 필요하다면 `pages/`에 새로운 마크다운 파일(`.md`)을 생성합니다.
4. `index.md`를 열고, 새로 생성되거나 업데이트된 문서를 적절한 카테고리에 링크합니다.
5. `log.md`를 열고 `## [YYYY-MM-DD] ingest | 문서제목` 형식으로 작업 로그를 추가합니다.

### 2.2 Query (질의 및 정보 종합)
사용자가 위키 내의 질문을 할 경우, AI는 다음 절차를 따릅니다.
1. `index.md`를 먼저 읽고 질문과 관련된 `pages/` 문서를 탐색합니다.
2. 탐색한 지식들을 조합하여 답변을 구성합니다.
3. 답변 과정에서 새로 도출된 유용한 비교나 분석, 인사이트가 있다면 이를 `pages/` 영역의 새로운 문서로 기록할지 판단하고, 적절하다면 문서를 생성 및 인덱스에 추가합니다.
4. `log.md`에 `## [YYYY-MM-DD] query | 질의 내용 요약` 형식으로 기록을 남깁니다.

### 2.3 Lint (위키 헬스체크 및 정리점검)
AI는 사용자의 요청에 따라 주기적으로 위키 무결성을 체크합니다.
1. 다른 문서들과 충돌하는 내용(Contradictions)이 없는지 확인합니다.
2. 어디에서도 링크를 걸지 않은 '고립된 문서(Orphan pages)'가 있는지 확인합니다.
3. 삭제되었거나 이름이 바뀐 문서를 가리키는 깨진 링크가 없는지 확인합니다.
4. `log.md`에 `## [YYYY-MM-DD] lint | 점검 내용 요약` 기록을 남깁니다.

## 3. Formatting Guidelines
- 문서 간 연결(Cross-reference)은 상대 경로 매핑을 활용해 진행합니다. (`[문서이름](문서경로.md)`)
- 모든 `pages/` 하위 생성 문서는 첫 줄에 최상위 제목(`# 문서 이름`)을 가집니다.
- 문서 하단 또는 관련 섹션에 반드시 **참고자료(Source)** 표시를 남깁니다. (예: `*Source: [원본이름](../raw/원본파일.md)*`)
